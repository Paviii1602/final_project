# 📱 NavBus APK Build Guide
## Convert to Android APK in 4 Steps

---

## Prerequisites (install once)

| Tool | Download | Why |
|------|----------|-----|
| Node.js 18+ | nodejs.org | Build React app |
| Android Studio | developer.android.com/studio | Build APK |
| JDK 17+ | Included with Android Studio | Java runtime |

---

## Step 1 — Deploy the Backend First

The APK needs a live backend URL. Deploy to Render (free):

1. Push the `backend/` folder to GitHub
2. Go to **render.com** → New Web Service
3. Settings:
   - Root Directory: `backend`
   - Build: `pip install -r requirements.txt`
   - Start: `gunicorn --worker-class eventlet -w 1 --bind 0.0.0.0:$PORT app:app`
   - Add env var: `SECRET_KEY` = any random string
4. Copy your URL: `https://navbus-xxxx.onrender.com`

---

## Step 2 — Configure the Frontend for APK

```bash
cd frontend

# Copy the APK env template
cp .env.apk .env

# Edit .env — replace the placeholder with your real backend URL
# VITE_BACKEND_URL=https://navbus-xxxx.onrender.com
# VITE_GOOGLE_MAPS_KEY=your_key  (optional)
```

Open `.env` and set:
```
VITE_BACKEND_URL=https://navbus-xxxx.onrender.com
```

---

## Step 3 — Build the React App

```bash
cd frontend
npm install
npm run build
```

This creates the `dist/` folder that Capacitor will package into the APK.

---

## Step 4 — Generate the APK with Capacitor

```bash
cd frontend

# Initialize Capacitor (first time only)
npx cap init NavBus com.navbus.app --web-dir dist

# Add Android platform (first time only)
npx cap add android

# Sync web build into Android project
npx cap sync android

# Open in Android Studio
npx cap open android
```

### In Android Studio:

1. Wait for Gradle sync to complete (~2 min)
2. Go to **Build → Build Bundle(s) / APK(s) → Build APK(s)**
3. Click **locate** when done → `app-debug.apk`

**The APK is at:**
```
frontend/android/app/build/outputs/apk/debug/app-debug.apk
```

### Transfer to Phone:
- Send via WhatsApp/email, or connect USB cable
- On phone: **Settings → Install unknown apps → Allow**
- Open the APK file to install

## After Any Code Changes

```bash
cd frontend

npx cap sync android   # sync to Android project
# Then rebuild in Android Studio
```

## APK Permissions Configured

The APK requests these permissions (set in `capacitor.config.json`):
- **Location** — for real-time GPS tracking (driver) and nearby buses
- **Internet** — to connect to the backend server
- **Network state** — to detect online/offline

## Troubleshooting

### "Cannot reach server" error in APK
→ Make sure `VITE_BACKEND_URL` is set in `.env` before building
→ The APK loads from `file://` so it can't use relative `/api/...` URLs

### "App not installed" when sideloading
→ Enable "Install from unknown sources" in Android settings
→ Uninstall any previous version first

### Map not showing
→ Add your Google Maps API key to `.env` as `VITE_GOOGLE_MAPS_KEY`
→ Restrict the key to your app's package name `com.navbus.app`

### Login "Invalid credentials"
→ Use the demo accounts: passenger/pass123 or driver/driver123
→ Or register a new account first

### GPS not working in APK
→ Grant location permission when prompted
→ Enable high accuracy location in phone settings

## Signing for Play Store (optional)

To publish to Google Play Store, you need a signed APK:

```bash
# Generate keystore (once)
keytool -genkey -v -keystore navbus.keystore -alias navbus -keyalg RSA -keysize 2048 -validity 10000

# In Android Studio: Build → Generate Signed Bundle/APK
# Use your keystore file
```
